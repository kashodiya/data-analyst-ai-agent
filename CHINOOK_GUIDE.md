# Chinook Database Guide for Data Analytics Agent

## Overview

The Chinook database represents a digital media store, including tables for artists, albums, media tracks, invoices, and customers. It's an excellent dataset for demonstrating the Data Analytics Agent's capabilities.

## Database Statistics

- **11 Tables** with rich relationships
- **3,503 Tracks** from various artists
- **347 Albums** across multiple genres
- **275 Artists** in the catalog
- **59 Customers** from 24 countries
- **412 Invoices** with 2,240 line items
- **8 Employees** in a hierarchical structure

## How to Connect

1. Open the Data Analytics Agent at `http://localhost:8080`
2. Click the database icon (🗄️) in the top bar
3. Enter connection string: `../data/chinook.db`
4. Select "sqlite" as database type
5. Click "Connect"

## Database Schema

### Core Business Entities

#### Music Catalog
- **Artist** - Music artists/bands
- **Album** - Albums released by artists
- **Track** - Individual songs/tracks
- **Genre** - Music genres (Rock, Jazz, etc.)
- **MediaType** - Format types (MP3, AAC, etc.)
- **Playlist** - Curated track collections
- **PlaylistTrack** - Tracks in playlists

#### Business Operations
- **Customer** - Store customers
- **Employee** - Store employees (sales support)
- **Invoice** - Customer purchases
- **InvoiceLine** - Individual items in invoices

## Key Relationships

```
Artist (1) ─── (N) Album
Album (1) ─── (N) Track
Track (N) ─── (N) Playlist (via PlaylistTrack)
Track (1) ─── (N) InvoiceLine
Genre (1) ─── (N) Track
MediaType (1) ─── (N) Track
Customer (1) ─── (N) Invoice
Invoice (1) ─── (N) InvoiceLine
Employee (1) ─── (N) Customer (as SupportRep)
Employee (1) ─── (N) Employee (ReportsTo)
```

## Tips for Best Results

1. **Start Simple**: Begin with basic queries to understand the data structure
2. **Use Natural Language**: The agent understands questions like "Show me the best customers"
3. **Ask for Explanations**: The agent will explain its SQL approach
4. **Iterate**: Based on results, ask follow-up questions for deeper insights
5. **Combine Metrics**: Ask for multiple metrics in one query for comprehensive views

## Common Aggregations

- **Revenue Metrics**: Total, Average, Min, Max by various dimensions
- **Count Analysis**: Number of tracks, albums, customers by category
- **Time Series**: Sales over time, growth rates, seasonal patterns
- **Rankings**: Top N by various metrics
- **Distributions**: Spread of data across categories

## Data Quality Notes

- All financial amounts are in USD
- Dates range from 2009 to 2013
- Some customers may have NULL values for company, state, or fax
- Track composers may be NULL for some entries
- All employees have a sales support role

## Troubleshooting

If you get connection errors:
- Ensure the database file exists at `data/chinook.db`
- Check the connection string is exactly: `../data/chinook.db`
- Verify the database type is set to "sqlite"
